# 📊 Guía de la Base de Datos Local (A+I-Planing)

Este documento detalla la configuración técnica de la base de datos PostgreSQL migrada desde Supabase, permitiendo que otras aplicaciones, scripts o agentes de IA consuman y consulten los datos de forma offline.

## 1. Detalles de Conexión

- **Motor**: PostgreSQL 16
- **Host**: `127.0.0.1` (localhost)
- **Puerto**: `5433`
- **Base de Datos**: `aiplaning_local`
- **Usuario**: `postgres`
- **Contraseña**: `Postgres2024!`
- **Modo de Autenticación**: `trust` (para conexiones locales)

### Cadena de Conexión (SQLAlchemy / Python)
```python
conn_str = "postgresql://postgres:Postgres2024!@127.0.0.1:5433/aiplaning_local"
```

## 2. Tipos de Tablas y Lógica de Sincronización

La base de datos se actualiza diariamente mediante los scripts en `backend/daily_sync.py` y se divide en dos categorías lógicas:

### 🔴 Tablas Incrementales (Transaccionales)
Estas tablas **acumulan** datos históricos. Solo se insertan registros nuevos que no existían previamente (usando firmas de duplicados).
- `sap_consumo_movimientos`: Movimientos de almacén (consumos, ingresos).
- `sap_produccion`: Órdenes de producción finalizadas.
- `ai_anomaly_alerts`: Alertas generadas por el motor de IA.

### 🔵 Tablas Snapshot (Maestras / Refresco Total)
Estas tablas representan una **foto del momento**. En cada sincronización, se truncan (vacían) y se vuelven a llenar con el inventario o plan actual.
- `sap_stock_mb52`: Inventario actual en tiempo real de SAP.
- `sap_programa_produccion`: Plan de producción para los próximos meses.
- `sap_maestro_articulos`: Catálogo de materiales.
- `sap_demanda_proyectada`: Demanda futura mensual.

## 3. Arquitectura de Sincronización y "Proxies"

Las tareas programadas de Windows (Windows Task Scheduler) ejecutan scripts en la raíz `D:\Base de datos\`. Para que estas tareas funcionen sin modificar la configuración de Windows, se han implementado **Proxies**:

1. **Proxy en Raíz**: `D:\Base de datos\run_daily.bat`
2. **Redirección**: Al ejecutarse, este archivo redirige el flujo al motor local en `D:\Base de datos\A+I-Planing\backend\run_daily.bat`.
3. **Escritura Local**: El motor local escribe en el puerto `5433` en lugar de subir datos a la nube.

## 4. Estructura Técnica y Esquema

- **IDs Autoincrementales**: Las tablas usan columnas `id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`.
- **Columnas de Auditoría**:
    - `created_at`: Fecha real de inserción en la base de datos.
    - `creado_el`: Fecha originaria extraída desde SAP (Excel).
    - `last_updated_at` / `updated_at`: Indicador de cuándo se refrescó el Snapshot.

## 5. Monitoreo (Dashboard)

Se puede visualizar el estado de la base de datos y los últimos registros sincronizados ejecutando:
```bash
cd backend
streamlit run dashboard.py
```
O mediante el acceso directo global `v_ejecutar_servidores.bat`.
